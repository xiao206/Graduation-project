**宿舍AA分摊助手（纯本地）测试用例**

# 1 测试方案

主要覆盖以下场景：

1. 成员管理：默认 6 人、改名、启用/停用；
2. 账单：新增/编辑/删除、参与人分摊、日期筛选；
3. 结算：范围选择、应收应付汇总计算、分摊余数处理；
4. 数据：导出复制、导入覆盖、非法数据拦截。

# 3 测试结论

系统在纯本地环境下能够完成成员管理、账单录入、平均分摊与结算汇总、数据备份恢复等核心功能。对关键异常输入（金额无效、非法 JSON）具备提示与保护机制，满足毕业设计阶段的功能完整性要求。
